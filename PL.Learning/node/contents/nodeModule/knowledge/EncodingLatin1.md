### [ISO-8859-1 (Latin-1)](#)
**ISO-8859-1**，通常被称为 **Latin-1** 或 **西欧语言**，是国际标准化组织（ISO）制定的 **ISO/IEC 8859** 系列标准中的第一个8位字符编码标准。它旨在为西欧语言提供一个统一的字符集，以支持除英语外，如法语、德语、西班牙语、葡萄牙语、意大利语、北欧语言等主要语言中常见的拉丁字母扩展字符。

### 核心特点

1.  **8位编码**：
    *   ISO-8859-1 使用单个字节（8位）来表示一个字符，因此总共可以表示 256 (2^8) 个不同的字符。
    *   这256个码位被分为两个部分：
        *   **0x00 - 0x7F (0 - 127)**：这部分与 **ASCII** (美国信息交换标准代码) 完全兼容。这意味着所有标准的英文字母（A-Z, a-z）、数字（0-9）、标点符号（如 , . ; : ! ?）以及控制字符（如换行符、制表符）在 ISO-8859-1 中的编码与 ASCII 相同。
        *   **0x80 - 0xFF (128 - 255)**：这部分是 ISO-8859-1 的扩展区域，包含了128个额外的字符。这些字符主要用于表示西欧语言中的重音字母、货币符号和一些特殊符号。

2.  **支持的语言**：
    *   主要覆盖了使用拉丁字母的西欧语言，包括：
        *   阿尔巴尼亚语
        *   布列塔尼语
        *   加泰罗尼亚语
        *   丹麦语
        *   荷兰语
        *   英语
        *   法罗语
        *   弗里西语
        *   加利西亚语
        *   德语
        *   格陵兰语
        *   冰岛语
        *   爱尔兰盖尔语（部分）
        *   意大利语
        *   拉丁语
        *   卢森堡语
        *   挪威语
        *   葡萄牙语
        *   罗马尼亚语（部分，缺少 Ș, Ț）
        *   苏格兰盖尔语
        *   西班牙语
        *   瑞典语
        *   土耳其语（部分，缺少 Ğ, İ, Ş, ı, ş）
        *   威尔士语

3.  **包含的字符示例**：
    *   **重音字母**：`À, Á, Â, Ã, Ä, Å, Æ, Ç, È, É, Ê, Ë, Ì, Í, Î, Ï, Ð, Ñ, Ò, Ó, Ô, Õ, Ö, Ø, Ù, Ú, Û, Ü, Ý, Þ, ß, à, á, â, ã, ä, å, æ, ç, è, é, ê, ë, ì, í, î, ï, ð, ñ, ò, ó, ô, õ, ö, ø, ù, ú, û, ü, ý, þ, ÿ`
    *   **货币符号**：`¤` (通用货币符号), `¥` (日元/人民币), `£` (英镑)
    *   **其他符号**：`¡` (倒感叹号), `¿` (倒问号), `©` (版权符号), `®` (注册商标符号), `°` (度符号), `±` (加减号), `µ` (微符号), `¶` (段落符号), `·` (中点)

### 历史地位与重要性

*   **Web 早期的默认编码**：在互联网和万维网发展的早期（1990年代至2000年代初），ISO-8859-1 是 **HTML 4.01** 和许多早期 Web 浏览器的默认字符编码。当网页未明确声明编码时，浏览器通常会假设其为 ISO-8859-1。
*   **广泛兼容性**：由于其与 ASCII 的兼容性以及对西欧语言的支持，它在很长一段时间内是处理西欧文本的事实标准，尤其是在电子邮件、文件传输和早期操作系统中。

### 与 UTF-8 的关系及现状

1.  **UTF-8 的优势**：
    *   **Unicode 兼容**：UTF-8 是 Unicode 的一种可变长度编码方式，可以表示世界上几乎所有的书写系统（包括中文、日文、阿拉伯文、希伯来文、西里尔字母等），而 ISO-8859-1 仅限于西欧语言。
    *   **ASCII 兼容**：UTF-8 的前128个字符（0x00-0x7F）也与 ASCII 完全兼容，这使得纯英文文本在 UTF-8 和 ASCII/ISO-8859-1 下是相同的。
    *   **无乱码风险**：UTF-8 能正确处理多语言混合的文本，而 ISO-8859-1 无法表示非西欧字符，强行显示会导致“乱码”。

2.  **ISO-8859-1 的局限性**：
    *   **无法表示非西欧字符**：这是其最大的缺点。它不能表示中文、日文、韩文、俄文、阿拉伯文等。
    *   **缺少部分西欧字符**：即使在西欧范围内，它也缺少一些字符，如罗马尼亚语的 `Ș` (S-comma) 和 `Ț` (T-comma)，或土耳其语的 `İ` (带点大写I)。
    *   **单字节限制**：8位编码限制了其字符集的扩展性。

3.  **当前状态**：
    *   **已被 UTF-8 取代**：现代 Web 标准（HTML5）和软件开发强烈推荐使用 **UTF-8** 作为首选编码。UTF-8 已成为互联网上最主流的字符编码。
    *   **遗留系统**：ISO-8859-1 仍然存在于一些旧的系统、数据库、文件或配置中。处理这些遗留数据时，可能仍需要识别和转换 ISO-8859-1 编码。
    *   **误解**：有时人们会误以为 `Content-Type: text/html; charset=ISO-8859-1` 可以“修复”乱码，但实际上，这只会将字节流按照 Latin-1 规则解释，如果原始数据是 UTF-8 或其他编码，这通常会导致更严重的乱码。

### 总结

**ISO-8859-1 (Latin-1)** 是一个重要的历史字符编码标准，它为西欧语言的数字化和早期互联网的发展奠定了基础。它与 ASCII 兼容，并包含了西欧语言所需的重音字符和符号。

然而，随着全球化和多语言需求的增长，其单字节、仅限西欧的局限性变得明显。如今，**UTF-8** 凭借其对 Unicode 的全面支持和向后兼容性，已经完全取代了 ISO-8859-1，成为现代应用、网页和数据交换的绝对主流和推荐标准。了解 ISO-8859-1 对于理解字符编码的历史和处理遗留系统仍然很有价值，但在新项目中应始终优先选择 UTF-8。
